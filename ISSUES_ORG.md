# 組織・業務課題一覧

Dental X-Ray Navigator の業務プロセス・法的要件に関する問題点。

---

## 🔴 緊急度：高（法的リスク）

### 1. 監査証跡（Audit Trail）がない
- **現状**: 誰が何を変更したか追跡する機能がない
- **リスク**: 医療法・個人情報保護法への抵触
- **影響**: 医療事故発生時の原因追跡不可、法的責任問題
- **対応**: 全操作のログ記録機能実装
- **関連法規**: 医療法、個人情報保護法

### 2. インフォームドコンセント記録なし
- **現状**: X線撮影のリスク説明・患者同意の記録機能がない
- **リスク**: 医療訴訟時の防御困難
- **影響**: 法的リスク、患者トラブル
- **対応**: 同意取得・記録フローの追加
- **関連法規**: 医療法第1条の4

---

## 🟠 優先度：高（業務プロセス）

### 3. 承認フロー（ワークフロー）がない
- **現状**: CT等ハイリスク検査も誰でも登録可能
- **リスク**: 医師の確認・承認なしで撮影実施
- **影響**: 医療事故リスク、過剰被曝
- **対応**: 撮影依頼→医師承認→撮影実施のワークフロー実装
- **運用例**:
  - 受付: 依頼作成
  - 医師: 承認/却下
  - 技師: 撮影実施

### 4. 権限管理（RBAC）がない
- **現状**: 全スタッフが全機能を操作可能
- **リスク**: 役割に応じた操作制限ができない
- **影響**: 誤操作、不正操作のリスク
- **対応**: ロールベースアクセス制御の実装
- **役割例**:
  | 役割 | 権限 |
  |------|------|
  | 医師（doctor） | 全機能、承認権限 |
  | 技師（technician） | 撮影実施、照射録入力 |
  | 受付（receptionist） | 依頼作成、患者登録 |

### 5. 診療報酬請求システムとの連携なし
- **現状**: 保険点数計算のみ実装。外部システムとの連携なし
- **リスク**: 請求漏れ、二重請求
- **影響**: 収益損失、コンプライアンス問題
- **対応**: レセプトシステムとの連携機能（CSV/API）
- **連携候補**: ORCA、レセプトコンピュータ

---

## 🟡 中期的課題（運用改善）

### 6. レポート・エクスポート機能なし
- **現状**: 統計ダッシュボードのみ、出力機能なし
- **リスク**: 月次報告、監査対応が手作業
- **影響**: 管理業務の負担
- **対応**: CSV/PDF出力、月次レポート自動生成

### 7. 通知・アラート機能なし
- **現状**: 予定時刻の通知、承認待ち通知がない
- **リスク**: 撮影漏れ、承認遅延
- **影響**: 患者待ち時間増加、業務効率低下
- **対応**: プッシュ通知、メール通知の実装

---

## 📋 業務フロー上の課題

### 現状のフロー（問題あり）
```
受付 → 依頼作成 → 即座に撮影可能 → 完了
         ↑
    医師確認なし
```

### あるべきフロー
```
受付 → 依頼作成 → 医師承認 → 撮影実施 → 照射録入力 → 完了
                    ↓
                  却下 → 修正依頼
```
