## 2. 背景と目的

### 2.1 背景

**現状の運用（dental enote 6）**
- 日付ごとに撮影依頼用テンプレートを作成
- 各ページで一件ずつ撮影内容を記載
- 放射線技師がそれを見ながら撮影

**現状の課題**
| 課題 | 詳細 |
|------|------|
| 一覧性が低い | 撮影待ち患者が一目で分からない |
| 操作が煩雑 | ページごとの記入が手間 |
| 集計ができない | 点数や実績の自動集計機能がない |
| 検索できない | 過去の記録を探しにくい |

### 2.2 目的
歯科医院のX線撮影業務を効率化し、dental enote 6の課題を解決する：

1. **撮影待ち患者の管理** - 待機中の依頼を一覧で把握できる
2. **照射録の記録・管理** - 素早く記録でき、後から検索可能
3. **保険点数の集計** - 自動計算・期間集計・エクスポート

### 2.3 ゴール
- [ ] 撮影依頼の抜け漏れをゼロにする
- [ ] 照射録の記録を1件10秒以内で完了できる
- [ ] 月次の点数集計をワンクリックで出力できる
- [ ] 過去の照射録を患者名・日付で即座に検索できる

## 7. 制約・前提条件

### 7.1 技術的制約
- フロントエンド: React + TypeScript
- バックエンド: Firebase Firestore
- ホスティング: Firebase Hosting

### 7.2 運用上の制約
- 1医院のみで使用（マルチテナント不要）
- インターネット接続が前提

### 7.3 前提条件
- 患者IDは既存システム（レセコン等）と同じものを使用
- 保険点数は診療報酬改定時に手動更新

## 8. 用語集

| 用語 | 説明 |
|------|------|
| 照射録 | X線撮影時の条件（kV/mA/sec）と担当者を記録したもの |
| 被曝テンプレート | 患者の年齢・体格に応じた照射条件の推奨値 |
| FDI歯番号 | 国際歯科連盟の歯番号システム（11-48） |
| BITEWING | 咬翼法撮影（左右それぞれ撮影） |
| PACS | 医用画像管理システム |
| レセコン | 診療報酬請求用コンピュータ |
